## Noted, not planned
Surfaced by the audit but below the leverage/confidence floor this pass —
kept so a later pass need not re-discover them:

- **`-auto` ↔ interactive fragment fork dedupe** — handle-bot-reviews-auto.md
  / watch-pipelines-auto.md restate their interactive references
  near-verbatim (~400 duplicated lines; the gh fetch calls + GraphQL
  query are byte-similar); drift already happened once (stability window
  PR #21 landed only on the auto side). 5–8 SP, medium risk; do after
  008 establishes the references/pipeline/ home.
- **process-tickets.md ↔ process-plans.md mirror enforcement** — the two
  orchestrators are documented as "word-for-word identical modulo the
  unit noun" but have drifted structurally; either extract the shared
  body or ship a normalize-and-diff check. Needs a normalization spec
  first; revisit after 002.
- **Split workflows.py (1,739 lines, ~7 concerns) / insights.py (1,301
  lines)** — real accretion debt, but splitting before tests exist is
  risk-inverted; revisit after 004 lands. The CLI entry paths must stay
  byte-stable (CONTRIBUTING §9.4).
- **`cmd_render` template semantics** — sequential str.replace lets
  output values re-expand `{{...}}` and shadow `project.*`/`run.*` keys;
  004 pins the current contract; a deliberate escaping design is future
  work.
- **`update-step`/`update-run` accept arbitrary keys** — typo'd keys
  write silently (workflows.py:802-845); a key allowlist would harden
  the state schema. Low blast radius today.
- **Unused roster agents (ceo, cto, engineering-manager, sre)** — zero
  consumers in bundled workflows/skills; documented as user-facing
  roster, so trim-or-justify is a product call, not cleanup.
- **Hardcoded stack-specific `allowed-tools`** (`vendor/bin/codecept`,
  `go`, `npm` in pr-watch/implement skills) — wrong-altitude allowlist
  in a tech-neutral plugin; dropping entries adds permission prompts
  mid-autonomous-run, needs a policy decision.
- **Insights ledger append/compact race** — concurrent SessionEnd append
  during `mine`'s compact can lose a record; self-heal mostly covers it;
  low confidence it bites in practice (single-user tool).
- **Redaction depth** — shape-based redaction misses short passphrases /
  internal hostnames (<28 chars); documented tradeoff; revisit if the
  ledger ever leaves the machine.
- **bats coverage for session-end-ingest.sh** — 004 covers the contract
  via pytest+subprocess instead; dedicated bats infra not justified yet.
- **pip `--require-hashes`** — stronger than 007's version ranges;
  breaks `--user` convenience installs; revisit if supply-chain posture
  tightens.
